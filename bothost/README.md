# bothost

Telegram-бот, в который пользователи присылают `.py`-файл со своим ботом, оплачивают подписку через **Telegram Stars** и получают запущенного бота. Платформа изолирует и запускает каждый пользовательский бот в отдельном Docker-контейнере, следит за подпиской и автоматически останавливает истёкшие.

## Тарифы (по умолчанию, можно править в `.env`)

| Тариф | ⭐ | Дней | Ботов |
|---|---|---|---|
| Старт | 50 | 14 | 1 |
| Плюс | 130 | 14 | 3 |
| Про | 200 | 14 | 5 |
| Без лимита | 400 | 14 | 20 |

Покупки складываются: квота берётся максимальной из имеющейся, дни добавляются.

## Возможности

- Несколько ботов на пользователя (по тарифу).
- Управление каждым ботом из удобного меню (▶️ старт / ⏹ стоп / 🔄 рестарт / 📜 логи / ✏️ переименовать / 🔁 заменить код / 🗑 удалить).
- Pre-built образ `bothost-userbot:latest` с предустановленными `aiogram`, `pyTelegramBotAPI`, `python-telegram-bot`, `requests`, `aiohttp`, `httpx` — пользовательские контейнеры стартуют за секунды.
- Изоляция: лимиты CPU/RAM, read-only корень, нет capabilities, отдельная сеть.
- Автоостановка по истечении подписки + уведомление пользователю.
- Админ-команды: статистика, список подписок, продление, принудительная остановка, ускорение истечения для тестов.
- Хранение в SQLite (один файл).

## Быстрый старт (Docker Compose)

1. Получите токен главного бота у [@BotFather](https://t.me/BotFather).
2. Скопируйте `.env.example` в `.env` и заполните `BOT_TOKEN`/`ADMIN_IDS`:

   ```bash
   cp .env.example .env
   $EDITOR .env
   ```

3. Поднимите проект:

   ```bash
   docker compose up -d --build
   ```

   Compose сначала соберёт образ для пользовательских ботов (`bothost-userbot:latest`), затем запустит главный бот.

4. Откройте бота в Telegram, нажмите `/start` и следуйте кнопкам.

│   ├── scheduler.py        # автоостановка при истечении
│   ├── validator.py        # проверка загружаемого .py
│   ├── keyboards.py        # inline-клавиатуры
│   ├── states.py           # FSM-состояния
│   └── handlers/
│       ├── start.py        # /start, /help, главное меню
│       ├── code.py         # загрузка .py, имя, замена кода
│       ├── payment.py      # тарифы + Stars

## Команды

### Пользователю

- `/start` — приветствие, главное меню.
- `/help` — инструкция и ограничения.
- `/buy` — выбрать тариф и оплатить.
- `/bots` (или `/mybot`) — список своих ботов с кнопками.
- `/status` — статус подписки и список ботов.
- Кнопки на каждом боте: ▶️ старт / ⏹ стоп / 🔄 рестарт / 📜 логи / ✏️ переименовать / 🔁 заменить код / 🗑 удалить.

### Администратору (`ADMIN_IDS`)

- `/stats` — пользователи, активные подписки, запущенные боты, всего звёзд.
- `/users` — список активных подписок.
- `/extend <tg_id> <дни> [квота]` — продлить подписку на N дней (опционально поднять квоту).
- `/admin_stop <tg_id|all>` — остановить боты пользователя (или всех).
- `/expire_in <tg_id> <секунд>` — заставить подписку истечь через N секунд (для тестов автоостановки).

## Безопасность

Пользовательский код выполняется в Docker-контейнере на базе `bothost-userbot:latest` (Python 3.12) с:

- `--memory 256m --cpus 0.5` — жёсткие лимиты;
- read-only корнем + `tmpfs /tmp` (64 МБ);
- без доступа к хост-сокету Docker и хост-файловой системе (кроме своего `/app`);
- без капабилити (`--cap-drop=ALL`, `no-new-privileges`);
- ограничение `pids_limit=128`;
- собственной сетью контейнера.

Главный бот не передаёт пользовательскому контейнеру никаких ENV-переменных — пишите токен и настройки прямо в `.py`.

> ⚠️ Запуск произвольного Python-кода всегда несёт остаточный риск (kernel exploits, side-channel). Не размещайте сервис на машине, где хранится что-то ценное; используйте отдельную VPS.
