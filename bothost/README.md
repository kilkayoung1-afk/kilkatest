# bothost

Telegram-бот, в который пользователи присылают `.py`-файл со своим ботом, оплачивают **50 ⭐ Telegram Stars** за неделю работы — и получают запущенного бота на 7 дней. Платформа сама изолирует и запускает каждый пользовательский бот в отдельном Docker-контейнере, следит за подпиской и автоматически останавливает истёкшие.

## Возможности

- Загрузка одного `.py` файла как Telegram-бота.
- Оплата подписки **50 ⭐ за 7 дней** через `sendInvoice` (валюта `XTR`).
- Изолированный запуск каждого пользовательского бота в отдельном Docker-контейнере с лимитами CPU/RAM.
- Автоматическая остановка контейнера при истечении подписки + уведомление пользователю.
- Команды управления своим ботом: запуск, перезапуск, остановка, просмотр логов.
- Админ-команды: статистика, список пользователей, ручное продление, ручная остановка.
- Хранение состояния в SQLite (один файл, без внешних БД).

## Быстрый старт (Docker Compose)

1. Получите токен главного бота у [@BotFather](https://t.me/BotFather) (отдельный бот, не пользовательский).
2. Скопируйте `.env.example` в `.env` и заполните:

   ```bash
   cp .env.example .env
   $EDITOR .env
   ```

   Минимально нужно:
   - `BOT_TOKEN` — токен главного бота;
   - `ADMIN_IDS=7119847306` — ваш Telegram ID (можно несколько через запятую);
   - остальное оставьте по умолчанию.

3. Запустите:

   ```bash
   docker compose up -d --build
   ```

4. Откройте бота в Telegram, нажмите `/start` и следуйте инструкциям.

## Структура

```
bothost/
├── docker-compose.yml      # запуск главного бота + том для пользовательских ботов
├── Dockerfile              # образ главного бота
├── .env.example            # пример конфигурации
├── pyproject.toml          # python-зависимости и метаданные
├── requirements.txt        # «плоский» список зависимостей для Docker
├── bothost/                # исходники главного бота
│   ├── __main__.py         # точка входа
│   ├── config.py           # настройки из .env
│   ├── db.py               # SQLite-хранилище подписок и ботов
│   ├── runner.py           # запуск/остановка пользовательских контейнеров
│   ├── scheduler.py        # планировщик истечения подписок
│   ├── validator.py        # проверка загружаемого .py файла
│   ├── keyboards.py        # inline-клавиатуры
│   └── handlers/           # хендлеры aiogram
│       ├── start.py        # /start, /help, главное меню
│       ├── code.py         # приём .py файла и запуск
│       ├── payment.py      # инвойс Stars + успешная оплата
│       ├── manage.py       # /mybot, /stop, /restart, /logs
│       └── admin.py        # /stats, /users, /extend, /admin_stop
└── data/                   # bind-mount: SQLite + загруженные .py
```

## Команды

### Для пользователей

- `/start` — приветствие, главное меню.
- `/help` — инструкция и ограничения.
- `/buy` — купить/продлить подписку (50 ⭐ за 7 дней).
- `/mybot` — статус активной подписки и бота.
- `/stop` — остановить своего бота (подписка не сгорает).
- `/restart` — перезапустить своего бота.
- `/logs` — последние 50 строк логов своего бота.

### Для администратора (`ADMIN_IDS`)

- `/stats` — общая статистика (пользователи, активные подписки, доход в звёздах).
- `/users` — список пользователей с активной подпиской.
- `/extend <tg_id> <дни>` — продлить подписку пользователя.
- `/admin_stop <tg_id>` — принудительно остановить чужого бота.

## Безопасность

Пользовательский код выполняется в отдельном Docker-контейнере на базе `python:3.12-slim` с:

- `--memory 256m --cpus 0.5` — жёсткие лимиты;
- read-only корнем + временным `tmpfs` на `/tmp`;
- без доступа к хост-сокету Docker и хост-файловой системе (кроме своего `/app`);
- без капабилити (`--cap-drop=ALL`);
- собственной сетью контейнера (без linux capabilities, без host network).

Главный бот добавляет в окружение пользовательского контейнера `BOT_TOKEN` (берётся из текста файла или из переменных окружения, которые пользователь объявит в коде сам).

> ⚠️ Запуск произвольного Python-кода всегда сопряжён с риском. Платформа уменьшает поверхность атаки, но не устраняет её полностью. Не запускайте сервис на машине, где хранятся чувствительные данные.
